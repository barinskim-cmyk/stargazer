# Stargazer — Пользовательские сценарии

Документ описывает все пути пользователя через приложение Stargazer (iOS Capacitor).

**Дата:** Март 2026
**Версия:** совпадает с MODULE_SPEC.md

---

## 1. Первый запуск (новый пользователь)

### 1.1 Авторизация

Приложение проверяет наличие сессии Supabase (`sb.auth.getSession()`).

**Ветвление:**

- **Есть userId в localStorage** → пропускаем auth, переходим к §1.2
- **Есть валидная сессия Supabase** → восстанавливаем userId, имя из email, переходим к §1.2
- **Нет сессии и нет userId** → показываем экран авторизации (`#auth-screen`)

На экране авторизации три варианта:

| Действие | Результат |
|----------|-----------|
| Email signup | Регистрация через Supabase, сохраняем userId, имя = часть email до @ |
| Email login | Вход в существующий аккаунт |
| Гостевой режим | `guestMode = true`, генерируется анонимный UUID |

После авторизации вызывается `hideAuth()` → `_proceedFromAuth()` → `_checkAndShowTutorial()`.

### 1.2 Проверка туториала

`_checkAndShowTutorial()` проверяет:

- `loadProgress().rating === 0` И `tutorialDone` не установлен → запуск интерактивного туториала (§2)
- Иначе → `showHomeScreen()` (§3)

## 2. Туториал (первая игра)

Семифазный интерактивный урок на canvas с 4 звёздами.

### Фазы

| Фаза | Описание | Действие пользователя |
|------|----------|----------------------|
| 0 | Появляются 4 звезды по одной (fade in) | Ожидание |
| 1 | «Нажмите на звезду» | Тап по любой звезде |
| 2 | «Нажмите на другую звезду чтобы соединить» | Тап по второй звезде → линия |
| 3 | Звездочёт делает свой ход | Автоматически |
| 4 | «Ваш ход — замкните фигуру» | Тап для соединения |
| 5 | Захват области → подсветка | Автоматически |
| 6 | Игра продолжается до конца | Свободная игра |

### Особенности

- `_tutorialMode = true`, `COUNT = 4`, `stargazerLevel = 1` (минимальная сложность AI)
- На ходах пользователя появляются контекстные подсказки (`_tutorialTooltip`)
- Пульсирующая подсветка целевых звёзд (`_tutorialPulseIdx`)
- После завершения → `tutorialDone = '1'` в localStorage
- Авто-сохранение первого созвездия (`_tutorialAutoSave = true`)
- Экран победы показывает специальный текст `victory_first_const` вместо стандартного

**Далее:** экран победы (§6.3) с авто-сохранением → `goToMenu()` → домашний экран (§3)

## 3. Домашний экран

### 3.1 Фоновая анимация

При показе домашнего экрана запускается `_startHomeBgAnim()`:
- Берутся реальные IAU-созвездия из `REAL_CONSTELLATIONS` (88 штук)
- Случайное созвездие рисуется на canvas, затем плавно свайпится влево/вправо к следующему
- Под созвездием — его локализованное название
- Автоматическая смена каждые 6 секунд + свайп вручную

### 3.2 Элементы управления

| Элемент | Действие |
|---------|----------|
| Имя игрока (верхняя строка) | → Профиль (§8) |
| Рейтинг (`score: N ✦`) | Информационное |
| Кнопка «Играть» | → Выбор длины → Старт игры (§5) |
| Иконка «Мои созвездия» ✦ | → Sky Hub (§7) |
| Иконка «Небо» ✦ | → Sky Hub (§7) |
| Иконка «Настройки» ⚙ | → Настройки (§9) |

### 3.3 Выбор длины партии

При нажатии «Играть» раскрывается пикер длины:

| Вариант | COUNT | Описание |
|---------|-------|----------|
| Быстрая | 5 | ~2 мин |
| Средняя | 8 | ~5 мин |
| Длинная | 12 | ~10 мин |
| Эпическая | 18 | ~20 мин |

Каждая карточка показывает canvas-превью с примерным расположением звёзд.

**Примечание:** для режима 2 игроков COUNT устанавливается через отдельный UI на стартовом экране.

## 4. Стартовый экран (ввод имён)

Показывается только в режиме 2 игроков (дуэль).

### Шаги

1. **Выбор режима** — vs AI или vs Игрок
2. **Выбор количества звёзд** (5 / 8 / 12 / 18)
3. **Ввод имён** — 1 или 2 поля в зависимости от режима
4. Кнопка «Начать» → `startGame()` (§5)

В режиме AI: имя Игрока 1 = `playerName` из localStorage, Игрок 2 = «Звездочёт».

## 5. Игровой процесс

### 5.1 Инициализация (`startGame`)

1. Сброс всех анимационных RAF, состояния предыдущей партии
2. `resetGameState()` — единый сброс игрового состояния (points, lines, polygons, scores, etc.)
3. Генерация `COUNT` случайных точек с соблюдением `MIN_DIST`
4. Анимация появления точек (stagger 120ms)
5. Определение уровня AI из ранга игрока (`rankToAiLevel()`)
6. Туториальный режим: если `rating === 0` и `tutorialDone` не задан → `COUNT = 4`, `stargazerLevel = 1`

### 5.2 Единый путь хода (`applyMove`)

Все ходы (и игрока, и AI) проходят через одну функцию `applyMove(a, b)`:

1. Звук `connect` + haptic
2. Анимация рисования линии (200ms)
3. Добавление ребра в граф (`lines.set`)
4. Проверка захвата области (`checkScore`)
5. Смена игрока (`currentPlayer = 1 - currentPlayer`)
6. Обновление UI, туториальных подсказок, hints
7. Проверка `isGameOver()` → если да → `endGame()` (§6)
8. Обновление полосы вероятности (DOM, CSS transition)
9. Если режим AI и ход Звездочёта → `setTimeout(makeAIMove, 800)`

### 5.3 Ход игрока (click handler)

1. **Тап по звезде** → звезда выделяется (пульсирующее кольцо)
2. **Тап по второй звезде** → валидация хода (`validate(a, b)`)
   - Валидный: `applyMove(a, b)` (§5.2)
   - Невалидный: пунктирная оранжевая линия + сообщение об ошибке

1. AI вызывается через `makeAIMove()` с задержкой 800ms (имитация раздумья)
2. Уровень сложности 1–10 определяет глубину оценки ходов:
   - Уровни 1–3: случайный выбор с базовым scoring
   - Уровни 4–7: учёт цепочек, блокирование оппонента
   - Уровни 8–10: полный просчёт, приоритет разрушения цепей оппонента
3. AI вызывает `applyMove(a, b)` (§5.2) — тот же путь, что и ход игрока

### 5.5 Элементы игрового UI

| Элемент | Описание |
|---------|----------|
| Верхняя панель | Очки, имена, индикатор хода |
| Полоса вероятности | Оценка шансов на победу (обновляется после каждого хода) |
| Кнопка «💡» | Включение/выключение подсказок |
| Кнопка «← Меню» | Выход в меню (`goToMenu()`) |
| Кнопка «👁» | Скрытие/показ панелей (расширение canvas) |

### 5.6 Подсказки (Hints)

Если включены (`_hintsOn`):
- Показываются до 5 лучших ходов пунктирными линиями
- `HIGH` — перспективный ход (яркий пунктир)
- `MEDIUM` — приемлемый ход (блёклый пунктир)
- В теме Stargazer — пульсирующая анимация подсказок

## 6. Завершение игры

`endGame()` вызывается когда `isGameOver()` возвращает `true` (больше нет валидных ходов).

### 6.1 Обработка результата

1. Подсчёт бонуса: победитель получает +1 очко
2. `progressionUpdate()` — обновление рейтинга, проверка достижений, проверка нового ранга
3. Генерация suprematist-компоновки (если выбрана эта тема)
4. Проверка rank-up → если новый ранг → церемония (§6.2)
5. Иначе → `finishEndGame()` (§6.3)

### 6.2 Церемония повышения ранга

Полноэкранная canvas-анимация (3.8 секунд):
- Тёмное звёздное небо (fade in 800ms)
- Текст «Новый ранг разблокирован»
- Название ранга крупным шрифтом + подранг (I/II/III)
- Эмблема ранга
- Задержка 2500ms → fade out → `finishEndGame()`

### 6.3 Экран результата (`finishEndGame`)

Три сценария:

#### Проигрыш (AI победил)
- Canvas полностью очищается
- Показывается `#loss-content` с итоговым счётом
- Кнопки: «Играть снова» и «← Выход»

#### Ничья
1. Canvas затемняется overlay (800ms)
2. Текст «Ничья» + счёт
3. Overlay исчезает, canvas уменьшается (`scale(0.42)`)
4. Появляется поле ввода имени созвездия
5. → сохранение (§6.4)

#### Победа
1. White flash + звёздные частицы + звук `victory` + haptic
2. Canvas затемняется и уменьшается (`scale(0.42)`, 1000ms)
3. Overlay исчезает — созвездие видно маленьким, но ярким
4. Появляется поле для имени созвездия (1600ms)
5. → сохранение (§6.4)

**Режим дуэли:** вместо одного canvas — два (left/right) с раздельными созвездиями и именами.

### 6.4 Сохранение созвездия

В зависимости от статуса пользователя:

| Статус | Кнопки |
|--------|--------|
| Туториал | Автоматическое сохранение, кнопки скрыты |
| Гость | Только «В мои созвездия» (localStorage) |
| Авторизован | «В мои созвездия» + «В небо ✦» (Supabase) |

#### Сохранение «В небо ✦»

1. Формируется объект constellation с координатами, линиями, полигонами
2. Сохранение в localStorage немедленно
3. Отправка в Supabase (или очередь при отсутствии сети)
4. Запуск `saveToSkyAnimation()`:
   - Небо появляется (400ms)
   - Каждая звезда летит к центру по дуге (1600ms) — каждая по своей траектории с перпендикулярным смещением
   - Точка мигает (1000ms)
   - Текст «Созвездие сохранено ✦» (400ms + 1000ms hold)
   - Fade out (600ms)
5. → `goToMenu()` → домашний экран

#### Сохранение «В мои созвездия»

1. Сохранение только в localStorage (без publishedAt)
2. Звук `twinkle` + haptic
3. Текст «Сохранено ✦»
4. Через 1500ms → `goToMenu()`

## 7. Sky Hub и навигация

### 7.1 Sky Hub (`#sky-hub-screen`)

Промежуточный экран навигации с двумя кнопками:

| Кнопка | Назначение |
|--------|------------|
| Мои созвездия | → Journey (§7.2) |
| Общее небо | → Sky Screen (§7.3) |

### 7.2 Journey — Мои созвездия (`#journey-screen`)

Сетка сохранённых созвездий в виде canvas-миниатюр.

- Загружается из `localStorage['saved_constellations']`
- Каждая карточка: мини-созвездие + название + дата
- Тап по карточке → Constellation Detail (§7.4)
- Обратно → Sky Hub или Home (в зависимости от `returnTo`)

### 7.3 Sky Screen — Общее небо (`#sky-screen`)

Бесконечное звёздное небо с созвездиями всех игроков.

- Данные из Supabase (realtime subscription)
- Панорамирование жестами (pan/zoom)
- Каждое созвездие — точка на небе; при приближении → видны линии и имя
- Фоновые звёзды генерируются детерминированно по хешу
- Тап по своему созвездию → Constellation Detail (§7.4)
- Оффлайн: показывается placeholder UI (`_skyShowOfflineUI`)

### 7.4 Constellation Detail (`#const-detail-screen`)

Подробный просмотр созвездия с canvas-рендерингом.

| Действие | Описание |
|----------|----------|
| Просмотр | Canvas с мерцающими звёздами, линиями |
| Переименование | Редактирование имени через input |
| Удаление | Подтверждение → удаление из localStorage |
| Публикация в небо | Если не опубликовано: Supabase upload + sky-animation |
| Поделиться | Генерация PNG через `_makeShareCanvas()` → native share API |
| Назад | → Journey Screen |

## 10. Оффлайн-режим

### Обнаружение

Слушатели `online` / `offline` на `window`.

### Поведение

- **Оффлайн-баннер:** Показывается при потере связи
- **Очередь сохранений:** Если Supabase недоступен → `_queueSkySave()` сохраняет payload в localStorage
- **Badge:** Число ожидающих синхронизации созвездий
- **Автосинхронизация:** При восстановлении связи → `_syncOfflineQueue()` отправляет всё в Supabase
- **Sky Screen:** Показывает offline UI placeholder

## 11. Визуальные темы в контексте сценариев

Тема влияет на все визуальные аспекты сценариев:

| Тема | Игровой canvas | Экран победы | Анимации |
|------|---------------|-------------|----------|
| Stargazer | Мерцающие звёзды, градиентные линии | Glow-эффекты, мерцание | Twinkle RAF loops |
| Minimal | Чистые линии, без эффектов | Минималистичная типографика | Нет анимаций |
| Suprematist | Геометрические блоки, виньетка | Детерминированная компоновка | Зерно + постобработка |
| Chalk | Меловой фон, рукописные линии | Chalk-style рисовка | Chalk animation RAF |

## 12. Схема основных переходов

```
Запуск приложения
    │
    ├── Есть сессия? ──да──→ _checkAndShowTutorial()
    │                              │
    │                      ├── Новичок? ──да──→ [Туториал] → [Победа + авто-save] → goToMenu()
    │                      │                                                              │
    │                      └── Нет ─────────→ showHomeScreen() ←──────────────────────────┘
    │
    └── Нет сессии, нет userId ──→ [Auth Screen]
                                       │
                                       ├── Email Login/Signup
                                       └── Гостевой режим
                                           │
                                           └── hideAuth() → _proceedFromAuth()
                                                                    │
                                                              (повтор сверху)

showHomeScreen()
    │
    ├── «Играть»        → Выбор длины → startGame() → [Игра] → endGame()
    │                                                              │
    │                                                    ├── Rank-up? → [Церемония] → finishEndGame()
    │                                                    └── finishEndGame()
    │                                                              │
    │                                               ├── Проигрыш → [Loss Screen] → Снова / Меню
    │                                               ├── Ничья    → [Name + Save] → goToMenu()
    │                                               └── Победа   → [Name + Save] → goToMenu()
    │
    ├── ✦ (Созвездия)   → [Sky Hub] ──→ Journey → [Detail] → Назад
    │                            └──→ Sky Screen → Назад
    │
    ├── ⚙ (Настройки)   → [Settings] → Назад
    │
    └── Имя игрока      → [Profile] → [Achievements] → Назад
```

## 14. Supabase-таблицы

| Таблица | Назначение | Ключевые поля |
|---------|------------|---------------|
| `constellations` | Опубликованные созвездия | `user_id`, `name`, `data` (JSON), `gx`, `gy` |

Realtime-подписка в sky-screen.js обновляет небо при появлении новых созвездий.
